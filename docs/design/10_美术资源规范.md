## 1. 概述

### 1.1 目的
本规范定义项目所需的所有美术资源的技术标准、视觉风格和交付要求，确保：
- 程序开发可以使用临时资源正常进行
- 外部美术制作者有明确的制作标准
- 最终资源可以无缝替换临时资源

### 1.2 资源使用策略
```
开发阶段使用临时资源 → 按规范制作正式资源 → 替换资源文件 → 无需修改代码
```

### 1.3 工业蒸汽朋克风格定位
- **主色调**: 铜色、黄铜色、深棕色、锈迹色
- **材质感**: 金属质感、机械感、齿轮纹理
- **光效**: 暖色调光源、电流发光效果
- **氛围**: 复古工业、精密机械、蒸汽时代

## 2. 技术规格总览

### 2.1 通用技术标准

| 规格项 | 要求 | 说明 |
|--------|------|------|
| **文件格式** | PNG (透明) / JPG (不透明) | 优先使用PNG |
| **色彩模式** | RGBA (32位) | 支持透明通道 |
| **分辨率** | 72 DPI | 屏幕显示标准 |
| **坐标系** | 左上角为原点 (0,0) | 符合游戏引擎标准 |
| **命名规范** | 小写字母+下划线 | 见命名规范章节 |

### 2.2 性能要求

| 性能指标 | 限制 | 原因 |
|---------|------|------|
| 单张图片最大尺寸 | 2048×2048 px | GPU纹理限制 |
| 单张图片文件大小 | ≤ 500 KB | 加载速度 |
| 总资源包大小 | ≤ 50 MB | 安装包体积 |
| 透明通道 | 仅必要时使用 | 性能考虑 |

## 3. 瓦片资源 (Tiles)

### 3.1 技术规格

| 规格项 | 数值 |
|--------|------|
| **尺寸** | 128×128 px（推荐）或 64×64 px |
| **格式** | PNG（带透明通道） |
| **内容区域** | 中心 80%（留边距避免裁切） |
| **旋转中心** | 图片中心点 (64, 64) |

### 3.2 瓦片类型清单

#### 必需资源（第一优先级）

| 文件名 | 类型 | 说明 | 尺寸 |
|--------|------|------|------|
| `tile_power_source.png` | 电源 | 发光效果，不可旋转 | 128×128 |
| `tile_terminal.png` | 终端 | 插座样式，不可旋转 | 128×128 |
| `tile_straight.png` | 直线导线 | 横向导线，可旋转 | 128×128 |
| `tile_corner.png` | 转角导线 | L形导线，可旋转 | 128×128 |
| `tile_empty.png` | 空白 | 底板纹理 | 128×128 |

#### 可选资源（第二优先级）

| 文件名 | 类型 | 说明 |
|--------|------|------|
| `tile_cross.png` | 十字交叉 | 四向导线 |
| `tile_t_junction.png` | T形交叉 | 三向导线 |

### 3.3 视觉设计要求

**电源 (Power Source)**:
- 设计风格：复古发电机或电池外观
- 必须元素：铜制外壳、指示灯/发光点
- 颜色：铜色主体 + 暖黄色光效
- 参考：蒸汽朋克风格的铜制机械装置

**终端 (Terminal)**:
- 设计风格：电路插座或接口
- 必须元素：插孔、金属触点
- 颜色：铜/黄铜色金属质感
- 状态：未连接时暗淡，连接时发光

**导线 (Straight/Corner)**:
- 设计风格：铜制管道或电缆
- 必须元素：明确的连接方向指示
- 颜色：铜色管道 + 暗色背景
- 细节：铆钉、接缝、锈迹（可选）

**空白瓦片 (Empty)**:
- 设计风格：金属底板纹理
- 颜色：深棕色/灰色
- 细节：划痕、磨损纹理（可选）

### 3.4 状态图层（可选增强）

为支持动态效果，可提供状态变体：

| 状态 | 命名格式 | 说明 |
|------|----------|------|
| 默认 | `tile_xxx.png` | 基础状态 |
| 通电 | `tile_xxx_powered.png` | 电流通过时 |
| 悬停 | `tile_xxx_hover.png` | 鼠标悬停 |
| 可点击 | `tile_xxx_clickable.png` | 可交互提示 |

### 3.5 临时资源说明

**开发阶段可使用**:
- 纯色色块 + 文字标注（如"POWER"、"LINE"）
- 简单几何图形（圆形、方形、线条）
- 开源资源库的占位图

**临时资源命名**:
- 添加 `_temp` 后缀，如 `tile_power_source_temp.png`
- 方便后续全局搜索替换

## 4. UI界面资源

### 4.1 按钮组件

#### 技术规格
- **尺寸**: 根据功能确定（见下表）
- **格式**: PNG（带透明通道）
- **状态**: 至少提供3个状态

#### 按钮清单

| 按钮类型 | 尺寸 (px) | 状态 | 文件命名 |
|---------|----------|------|----------|
| 主按钮 | 200×60 | 正常/悬停/按下 | `button_primary_[state].png` |
| 次按钮 | 160×50 | 正常/悬停/按下 | `button_secondary_[state].png` |
| 返回按钮 | 80×80 | 正常/悬停/按下 | `button_back_[state].png` |
| 暂停按钮 | 60×60 | 正常/悬停/按下 | `button_pause_[state].png` |
| 关卡选择 | 120×120 | 正常/悬停/已完成/锁定 | `button_level_[state].png` |

**状态后缀**:
- `_normal` - 正常状态
- `_hover` - 鼠标悬停
- `_pressed` - 按下状态
- `_disabled` - 禁用状态
- `_completed` - 已完成（关卡）
- `_locked` - 锁定（关卡）

#### 设计要求
- 风格：金属按钮、铆钉装饰
- 状态差异：明显的视觉反馈
- 文字：如需内嵌文字，使用英文占位
- 圆角：适度圆角（5-10px）

### 4.2 面板与背景

#### 技术规格

| 资源名称 | 尺寸 | 说明 |
|---------|------|------|
| 主菜单背景 | 1920×1080 | 或更高分辨率 |
| 游戏背景 | 1920×1080 | 暗色调，不抢眼 |
| 对话框面板 | 800×600 | 九宫格切片 |
| HUD背景 | 可变 | 半透明 |

#### 九宫格切片说明
对于可伸缩面板，提供九宫格切片资源：
- 四个角：固定大小
- 四条边：可横向/纵向拉伸
- 中心：可双向拉伸

**命名格式**: `panel_dialog_[position].png`
- 位置：`tl`(左上), `t`(上), `tr`(右上), `l`(左), `c`(中), `r`(右), `bl`(左下), `b`(下), `br`(右下)

### 4.3 图标资源

#### 技术规格
- **尺寸**: 64×64 px 或 32×32 px
- **格式**: PNG（带透明通道）
- **风格**: 线条清晰，识别性强

#### 图标清单

| 图标名称 | 用途 | 文件名 |
|---------|------|--------|
| 设置图标 | 设置按钮 | `icon_settings.png` |
| 音量图标 | 音量控制 | `icon_volume.png` / `icon_mute.png` |
| 暂停图标 | 暂停按钮 | `icon_pause.png` |
| 播放图标 | 继续按钮 | `icon_play.png` |
| 返回图标 | 返回上一级 | `icon_back.png` |
| 帮助图标 | 帮助说明 | `icon_help.png` |
| 关闭图标 | 关闭窗口 | `icon_close.png` |

## 5. 特效资源

### 5.1 粒子特效

#### 技术规格
- **尺寸**: 32×32 px 或 64×64 px
- **格式**: PNG（带透明通道）
- **颜色**: 支持程序着色

#### 特效清单

| 特效名称 | 用途 | 文件名 | 尺寸 |
|---------|------|--------|------|
| 火花粒子 | 电流效果 | `particle_spark.png` | 32×32 |
| 光晕 | 发光效果 | `particle_glow.png` | 64×64 |
| 星星 | 胜利特效 | `particle_star.png` | 32×32 |
| 光点 | 电流流动 | `particle_dot.png` | 16×16 |

### 5.2 精灵序列帧动画（可选）

如需复杂动画，提供序列帧：

**命名格式**: `animation_[name]_[frame].png`
- 示例：`animation_current_flow_001.png`, `animation_current_flow_002.png`...

**技术要求**:
- 帧数：8-16帧（避免过多）
- 帧尺寸：统一（如 128×128）
- 帧率：目标 12 FPS（可调）
- 循环：支持无缝循环

### 5.3 Shader效果（高级，可选）

对于发光、扫光等效果，可提供Shader实现指导：
- 发光Shader：边缘高光 + 内发光
- 电流流动：UV动画 + 遮罩

## 7. 资源命名规范

### 7.1 命名规则

**格式**: `[类别]_[名称]_[状态]_[序号].扩展名`

**规则**:
- 使用小写字母
- 单词间用下划线 `_` 分隔
- 不使用空格、中文、特殊符号
- 序号使用三位数字（001, 002...）

### 7.2 类别前缀

| 前缀 | 类别 | 示例 |
|------|------|------|
| `tile_` | 瓦片 | `tile_power_source.png` |
| `button_` | 按钮 | `button_primary_normal.png` |
| `icon_` | 图标 | `icon_settings.png` |
| `bg_` | 背景 | `bg_main_menu.jpg` |
| `panel_` | 面板 | `panel_dialog_c.png` |
| `particle_` | 粒子 | `particle_spark.png` |
| `animation_` | 动画 | `animation_current_flow_001.png` |
| `effect_` | 特效 | `effect_glow.png` |

### 7.3 命名示例

```
✅ 正确命名:
tile_power_source.png
tile_straight.png
button_primary_hover.png
icon_settings.png
bg_game_board.jpg
particle_spark_001.png

❌ 错误命名:
PowerSource.png              # 不要使用大驼峰
tile-straight.png            # 不要使用连字符
按钮_主要.png                # 不要使用中文
button primary hover.png     # 不要使用空格
tile_1.png                   # 名称太模糊
```

## 12. 资源替换流程

### 12.1 替换步骤

```
1. 确认新资源符合规范
2. 使用相同文件名覆盖临时资源
3. 清除游戏资源缓存（如有）
4. 运行游戏测试
5. 验证显示效果
6. 提交到版本控制
```

### 12.2 注意事项

**✅ 正确做法**:
- 保持文件名一致
- 保持尺寸一致
- 保持格式一致
- 批量替换后全面测试

**❌ 避免错误**:
- 不要修改文件名
- 不要改变尺寸
- 不要改变文件格式
- 不要只替换部分资源

## 14. 常见问题

### Q1: 为什么要用128×128而不是64×64？
**A**: 128×128提供更好的视觉质量和细节，同时兼容缩放。如果性能有限，可以降至64×64。

### Q2: 可以使用JPG代替PNG吗？
**A**: 不透明的背景可以用JPG，但瓦片和UI组件必须用PNG，因为需要透明通道。

### Q3: 如何制作九宫格面板？
**A**: 将面板切成9块，四角固定，边和中心可拉伸。网上有许多教程可参考。

### Q4: 临时资源可以用多久？
**A**: 整个开发阶段都可以使用，直到正式资源制作完成。代码不依赖资源内容，只依赖文件名和尺寸。

### Q5: 字体有版权问题吗？
**A**: 必须使用商用免费字体或购买授权。推荐Google Fonts中的开源字体。
